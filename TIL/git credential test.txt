git credential test
1. 기존 자격 삭제 후 push해보자
> 사용자 인증 팝업이 떴다.
browser 로그인을 하여 인증 후 push 가 완료 됐고, 
윈도우 자격 증명에는 저장이 되어있지 않았다.

2. 다시 push 해보자. 이번엔 토큰을 넣어볼까.
> 어 아무 것도 안묻고 바로 push 가 되어버리네?
 확인해봤더니, 1 push 후 자격증명 새로고침을 안해봤었구나 ;;; ㅋ
새로고침을 누르니 저장이 되어있었다. 
심지어 1 브라우저 로그인 후 로그아웃도 안한채 창만 닫아놨었어...
전후 인과관계 확인이 애매하니 다시 test 해봐야겠다.

3. 자격 증명 삭제, 모든 브라우저 깃헙 로그아웃 후 push.
> 브라우저 로그인으로 증명. push 완료. 자격증명 새로고침하니 저장됐음을 확인.

4. 모든 브라우저 깃헙 로그아웃 후 다시 push 해보자.(3에서 저장된 자격증명은 남아있는 상태)
>결과: 인증 요청 없이 바로 push 되었음.

5. 결론:
역시 처음 한번만 인증하면 되는 거 였고, 자격증명이 OS에 저장되어 이후 묻지 않는 것이며,
윈도우 자격 증명 발행시에는 브라우저 로그인이라고 해서 1회성은 아니었던 모양이다.
